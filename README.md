# Solock

`Solock` 是一个面向 Windows 的自动化控制程序。它会按照预设时间表持续运行，并在不同时间段自动执行以下动作：

- 维持系统唤醒，避免系统休眠。
- 监听默认音频输出设备变化，并把系统音量调整到预设值。
- 自动开启 Windows 移动热点，并在晚间切换到指定 SSID。
- 针对指定进程动态下发 WFP 出站拦截规则，控制这些程序是否可以联网。
- 在午间空闲达到阈值时直接关机。
- 在晚间空闲达到阈值时锁屏并关闭显示器。
- 首次运行时为当前用户自动注册“登录后自启动”的计划任务。

本 README 根据当前仓库中的源码实现整理，核心代码位于 `main.cpp`、`SolockController.h`、`SolockController.cpp`。

## 1. 程序功能总览

### 1.1 启动阶段

程序启动后会执行以下步骤：

1. 初始化 WinRT 多线程 Apartment。
2. 创建 `SolockController`。
3. 默认自动注册一个计划任务，使当前用户登录后自动启动本程序。
4. 初始化音频输出设备监听。
5. 清理旧的应用断网规则，确保程序启动时目标应用默认可联网。
6. 根据当前时间判断程序所处阶段，并立即应用对应策略。

在 Release 模式下，主程序进入 `Run()`，随后长期驻留运行。  
在 Debug 模式下，主程序进入 `RunAllFeaturesAcceleratedDebug()`，执行一轮加速的功能自检流程。

### 1.2 常驻心跳循环

进入主循环后，程序会持续重复以下逻辑：

- 每轮重新判断当前处于哪个时间阶段。
- 强制声明“系统保持唤醒”状态。
- 根据当前阶段修正系统主音量。
- 根据当前阶段执行热点、断网、关机、锁屏等动作。
- 等待下一次心跳，默认每 15 秒运行一次。

如果检测到默认音频输出设备发生变化，程序不会等完整的心跳周期，而是立即提前唤醒并重新设置音量。

## 2. 三个时间阶段

程序内部把一天划分为三个阶段：

| 阶段 | 默认时间 | 行为 |
| --- | --- | --- |
| 定时管控阶段 `ScheduledBlocks` | 除午间和晚间外的大部分时间 | 保持热点可用；按若干时间点对指定进程进行“定时断网/恢复联网”；音量保持正常值 |
| 午间空闲关机阶段 `MiddayIdleShutdown` | 12:10 - 12:50 | 保持热点可用；确保目标进程恢复联网；若用户空闲达到阈值则立即关机；音量降低 |
| 晚间后置阶段 `EveningPostAction` | 17:40 之后 | 强制断开指定进程的联网；将热点 SSID 切换到指定名称；若用户空闲达到阈值则锁屏并关闭显示器；音量降低 |

阶段判断规则如下：

- 如果当前时间大于等于晚间开始时间，则进入晚间后置阶段。
- 否则，如果当前时间位于午间时间窗内，则进入午间空闲关机阶段。
- 否则，进入定时管控阶段。

## 4. 定时断网功能

### 4.1 断网方式

程序不是通过关闭网卡或关闭整个网络来实现断网，而是使用 Windows Filtering Platform（WFP）为指定应用安装动态出站拦截规则：

- 使用 `FwpmEngineOpen0` 打开动态 WFP Session。
- 注册自定义 Sublayer。
- 用 `FwpmGetAppIdFromFileName0` 获取目标可执行文件的 `ALE_APP_ID`。
- 分别在 IPv4 和 IPv6 的 `ALE_AUTH_CONNECT` 层安装 `FWP_ACTION_BLOCK` 过滤器。

也就是说，程序控制的是“指定进程是否能发起出站连接”，而不是整台机器是否联网。

### 4.2 断网触发规则

在定时管控阶段，程序会检查当前时间是否落入以下任意时间窗：

- 08:40 - 08:50
- 09:30 - 09:40
- 10:20 - 10:30
- 11:10 - 11:20
- 15:10 - 15:20
- 16:00 - 16:10
- 16:50 - 17:00

若当前时间落入上述时间窗之一：

- 对目标进程安装 WFP 出站阻断规则。

若当前时间不在上述时间窗内：

- 删除动态 WFP 会话，恢复目标进程联网能力。

### 4.3 断网对象的解析方式

程序会遍历当前系统进程快照，仅查找与配置名匹配的可执行文件名，然后：

- 通过 `OpenProcess(PROCESS_QUERY_LIMITED_INFORMATION)` 打开进程；
- 通过 `QueryFullProcessImageNameW` 解析出完整路径；
- 以完整路径作为 WFP 规则绑定对象。

这意味着：

- 只有“当前确实正在运行”的目标进程才会被加入阻断列表；
- 如果某个目标进程未运行，则本轮不会为它创建新的过滤器；
- 如果某一路径已经创建过过滤器，那么后续从同一路径再次启动的实例，在过滤器被清理前仍会继续被阻断；
- 如果前一轮已经安装过过滤器，而这轮没有再找到进程，程序会保留现有过滤器，避免目标程序在短暂重启时逃逸联网。

### 4.4 断网规则的清理

WFP 规则由动态 Session 承载。程序会在以下时机主动清理：

- 启动时，先执行一次“确保可联网”；
- 定时窗口结束时；
- 程序析构时；
- Debug 的单独断网测试结束时。

## 5. 热点管理功能

### 5.1 启动热点

程序通过 `Windows.Networking.NetworkOperators::NetworkOperatorTetheringManager` 控制 Windows 移动热点。

常规阶段会执行“前置热点保障”：

- 如果当前热点 SSID 不是晚间指定 SSID，会把当前 SSID 记录为“原始 SSID”；
- 如果本地状态文件中保存过原始 SSID，且当前 SSID 与其不同，则优先恢复原始 SSID；
- 最终确保热点处于开启状态。

### 5.2 晚间切换热点 SSID

晚间阶段会执行以下动作：

- 如果当前 SSID 不是 `CMCC-24dF`，先把当前 SSID 保存为原始 SSID；
- 再把热点配置切换到 `CMCC-24dF`；
- 最后确保热点处于开启状态。

### 5.3 热点密码处理

当程序重配热点 SSID 时，会复用当前热点密码。  
如果读取到的密码长度不足 8 位，则回退为 `12345678`，以满足热点密码长度要求。

### 5.4 原始 SSID 状态文件

程序会把“切换前的热点名称”保存在：

`%LocalAppData%\Solock\original_hotspot_ssid.txt`

用途：

- 晚间切换到指定 SSID 前，记录原始热点名称；
- 白天再次进入普通阶段时，尽量恢复到原始热点名称。

## 6. 音量控制功能

程序会监听默认音频输出设备变化，并把系统主音量自动调整到目标值。

规则如下：

- 定时管控阶段：音量调到 `60%`
- 午间空闲关机阶段：音量调到 `35%`
- 晚间后置阶段：音量调到 `35%`

实现细节：

- 使用 `IMMDeviceEnumerator` 获取默认渲染设备；
- 使用 `IAudioEndpointVolume` 读取/设置系统主音量；
- 使用自定义 `IMMNotificationClient` 监听默认输出设备变化；
- 当检测到设备切换时，通过事件对象提前唤醒主循环，重新设置音量。

如果当前系统没有默认输出设备，程序会把这视为“无需调整”，而不是报错退出。

## 7. 空闲检测、关机、锁屏与灭屏

### 7.1 空闲检测

程序使用 `GetLastInputInfo()` 读取最后一次用户输入时间，并以 `GetTickCount64()` 计算空闲时长。

默认空闲阈值为 10 分钟。

### 7.2 午间空闲关机

在 `12:10 - 12:50` 期间：

- 程序会确保目标进程恢复联网；
- 程序会确保热点处于可用状态；
- 一旦检测到用户已连续空闲 10 分钟，即执行立即关机。

关机实现分两层：

1. 先申请 `SE_SHUTDOWN_NAME` 权限并调用 `ExitWindowsEx(EWX_POWEROFF | EWX_FORCEIFHUNG, ...)`；
2. 若失败，则回退到启动 `shutdown.exe /s /f /t 0 /d p:0:0`；
3. 若仍失败，则每秒重试，直到成功。

### 7.3 晚间空闲锁屏和灭屏

在 `17:40` 之后：

- 程序会持续保持指定进程断网；
- 程序会持续保持热点切换到晚间 SSID；
- 若用户连续空闲达到 10 分钟，则调用 `LockWorkStation()` 锁屏；
- 锁屏约 1.2 秒后，向所有窗口广播 `SC_MONITORPOWER`，关闭显示器；
- 该动作在一次晚间周期内只会成功执行一次，避免重复锁屏。

一旦程序离开晚间阶段，内部标记会被清零，下一次再次进入晚间阶段后可重新触发。

## 8. 保持系统唤醒

程序每轮都会调用：

`SetThreadExecutionState(ES_CONTINUOUS | ES_SYSTEM_REQUIRED)`

作用是持续声明“系统需要保持运行”，避免系统因为空闲而自动睡眠。  
程序退出时会调用 `SetThreadExecutionState(ES_CONTINUOUS)` 清理该状态。

## 9. 启动网络稳定等待

仅在以下条件下，程序会在启动初期执行“网络稳定等待”：

- 程序启动时当前阶段属于定时管控阶段。

等待规则：

- 连续检测网络是否具有 `InternetAccess`；
- 需要连续稳定 8 秒才视为通过；
- 最长等待 45 秒；
- 若中途某秒网络不可用，则稳定计数归零重新累计。

这个等待逻辑的目的是让程序在刚开机、刚连网、网络切换中的场景下稍微稳定后再继续执行。

## 10. 自动注册计划任务

默认情况下，程序启动时会自动创建或更新一个计划任务：

- 任务名：`Solock AutoStart`
- 触发器：当前用户登录时触发
- 运行身份：当前交互登录用户
- 运行级别：最高权限
- 多实例策略：忽略新实例
- 无执行时限：`PT0S`
- 动作：启动当前可执行文件，并把工作目录设为当前 EXE 所在目录

这样做的效果是：

- 用户登录后程序可自动启动；
- 任务会始终指向当前 EXE 路径；
- 再次运行程序时会覆盖更新任务定义。

## 13. 需要注意的实际行为

1. Release 模式会长期常驻，并真的执行计划任务注册、热点控制、WFP 断网、关机、锁屏和灭屏。
2. 程序只会为“当前已解析到完整路径的目标进程”创建过滤器，但同一路径后续新启动的实例在规则被清理前也会继续被阻断。
3. 晚间阶段开始后，目标进程会被持续断网，而不是只断 10 分钟。
4. 午间阶段不会断网目标进程，反而会强制恢复其联网能力。
5. 程序没有独立配置文件，当前所有策略都直接写死在 `SolockController::Options` 的默认值里。
